SIEM-Based Threat Monitoring

Real-time log collection, custom threat detection rules, and a live SIEM dashboard — giving you full visibility into what is happening across your network at all times.

Wazuh SIEM MITRE ATT&CK Real-Time Alerts NIST CSF Multi-Source Logs

Service Overview

Small organisations are increasingly targeted by cyberattacks but have no mechanism to detect threats in real time. The SIEM-Based Threat Monitoring service deploys and configures a fully functional Wazuh SIEM instance capable of ingesting logs from multiple endpoints simultaneously.

Custom detection rules are written to identify brute-force login attempts, new user account creation, suspicious process execution, and lateral movement — the most common techniques used by attackers in real engagements. Every rule is mapped to the MITRE ATT&CK framework, giving a standardised view of which adversary techniques are being detected.

The resulting dashboard presents live alert panels grouped by severity, top source IPs, and techniques fired — providing exactly the visibility a Security Operations Centre analyst needs to respond to incidents quickly.

Vulnerability Assessment

Systematic discovery and exploitation-risk scoring of every vulnerability across the target environment — producing a professional, client-ready report with prioritised remediation steps.

NmapOpenVASCVSS v3.1NIST CSFPDF Report

Service Overview

The Vulnerability Assessment service provides a structured, evidence-based view of an organisation's security weaknesses. Beginning with Nmap network discovery to map all live hosts and open ports, the service escalates to a full authenticated OpenVAS scan that identifies known vulnerabilities across services, operating systems, and configurations.

Every finding is assigned a CVSS score and categorised as Critical, High, Medium, or Low — allowing the organisation to prioritise remediation effort where it matters most. The top five critical findings receive detailed remediation guidance, giving the team a clear action plan from day one.

The final deliverable is a professional PDF report structured with an executive summary for leadership and a full technical findings section for the IT team — mirroring what a junior security consultant would produce for a real client engagement.

Firewall & Network Hardening

Identifying unnecessary network exposure, tightening firewall rules using CIS-recommended practices, and delivering a measurable before-and-after comparison that proves the security improvement.

iptablespfSenseCIS BenchmarksNmapAttack Surface Reduction

Service Overview

In many environments, systems run multiple open ports and services that are not required for normal operation. These unnecessary services silently expand the attack surface and can be exploited by attackers without any sophisticated technique. The Firewall & Network Hardening service addresses this directly.

The process begins with an Nmap baseline scan to establish the current security posture — every open port and running service documented as the starting point. The existing firewall configuration (pfSense or iptables) is then reviewed to identify overly permissive rules that allow unnecessary traffic into the system.

Hardening is applied using CIS (Center for Internet Security) Benchmark recommendations — closing unused ports, disabling unnecessary services, and tightening rules so that only essential network traffic is allowed. A second Nmap scan then verifies the improvement, and a before-and-after comparison report makes the reduction in attack surface tangible and reportable to leadership.

Endpoint Detection & Response

A lightweight EDR agent on every managed device — continuously monitoring behaviour, detecting threats that antivirus misses, automatically containing incidents, and providing full forensic investigation capability.

EDRBehavioural DetectionMITRE ATT&CKZero-DayForensicsAuto-Containment

Service Overview

Traditional antivirus solutions only detect known threats — leaving organisations exposed to zero-day exploits, fileless malware, and sophisticated lateral movement. The Endpoint Detection and Response service provides continuous, intelligence-driven monitoring across every managed device.

A lightweight EDR agent is deployed on all endpoints — laptops, desktops, servers, and mobile devices — feeding continuous telemetry into a cloud-native detection platform. The platform uses AI-driven behavioural analysis correlated against global threat intelligence and the MITRE ATT&CK framework to identify suspicious activity before it causes damage.

When threats are detected, the system automatically contains compromised endpoints, terminates malicious processes, and rolls back changes within seconds. Complex incidents are escalated for human-led investigation, supported by a full forensic attack timeline that shows exactly what happened, when, and how — essential for both incident response and compliance reporting.

// About the Project

What is SecureWatch?

A student-built cybersecurity service platform demonstrating professional-grade security engagements using industry-standard open-source tools — built entirely in a controlled lab environment.

The Project

SecureWatch is a capstone cybersecurity service platform built to demonstrate that small and medium-sized organisations can achieve real, professional-grade protection without enterprise-level spending.

The platform is built on industry-standard open-source tools including Wazuh, Nmap, and OpenVAS, deployed in a controlled lab environment to simulate a real client security engagement from start to finish.

The deliverables mirror what a junior security consultant would produce professionally — vulnerability reports with CVSS scoring, live SIEM dashboards with MITRE ATT&CK-mapped detection rules, firewall hardening reports, and full EDR documentation.

All work is performed within a controlled lab environment. No external or production systems are scanned or monitored. All tools used are open-source or free-tier.
